
# 前提知識：AWS SESの「サンドボックス」とは？

AWSアカウントを作成してすぐのSESは「サンドボックス（砂場）」という制限モードに入っています。このモードには以下の重要な制限があります。

- 送信元として許可したメールアドレスしか使えない
- 送信先（宛先）も、許可したメールアドレスにしか送れない  
    ※つまり、テスト段階では「自分のメアドから、自分のメアド（または開発メンバーのメアド）へ送る」ことしかできません
- 1日あたり200通まで

---

# 設定手順

## 手順1：リージョン（地域）の選択

1. AWSマネジメントコンソールにログイン
2. 画面上の検索バーで「SES」と入力し、「Amazon Simple Email Service」を選択
3. 【重要】画面右上でリージョンを選択
    - 日本国内向けのアプリであれば「アジアパシフィック（東京） ap-northeast-1」を選択

---

## 手順2：メールアドレス（ID）の登録と認証

1. アプリが送信元として使うメールアドレス（例: [info@myapp.com](mailto:info@myapp.com) や開発者の個人のGmailなど）を登録
2. SESの左メニューから **[ID (Identities)]** をクリック
3. **[ID の作成 (Create identity)]** ボタンをクリック
4. 以下を設定
    - ID の種類: 「メールアドレス」
    - メールアドレス: テストに使うメールアドレス
5. **[ID の作成]** をクリック

---

## 手順3：メールの受信確認（検証）

1. 登録したメールアドレスの受信トレイを確認
2. AWSから「Amazon Web Services – Email Address Verification Request」という件名のメールが届いている
3. メール内のリンクをクリック
4. ブラウザで「Congratulations!」と表示されれば認証完了
5. SES画面に戻り、ステータスが「検証済み (Verified)」になっていることを確認

**重要（サンドボックスのルール）**

- テスト送信をする宛先が送信元と異なる場合は、手順2〜3を繰り返して宛先も「検証済み」にする必要あり
- 自分から自分に送るテストなら1つだけでOK

---

## 手順4：コンソールからテスト送信

1. [ID] 一覧画面で検証済みのメールアドレスにチェック
2. 画面右上の **[テストメールの送信 (Send test email)]** をクリック
3. 以下を設定
    - フォーマット: フォーマット済み (Formatted)
    - シナリオ: カスタム (Custom)
    - カスタム受信者: 検証済みのメールアドレス（自分宛てなど）
    - 件名 / 本文: 適当に入力（例: テストです）
4. **[テストメールの送信]** をクリック
5. 受信トレイにメールが届けばSES側の準備完了

---

## 手順5：アプリ用のSMTP情報の取得

1. SESの左メニューから **[SMTP 設定 (SMTP settings)]** をクリック
2. 表示されているSMTPエンドポイント（例: `email-smtp.ap-northeast-1.amazonaws.com`）をメモ（サーバー名）
3. **[SMTP 認証情報の作成 (Create SMTP credentials)]** をクリック
4. IAMユーザー名の作成画面 → そのまま **[作成 (Create)]** をクリック
5. 「認証情報のダウンロード」画面が表示される
    - SMTP ユーザー名
    - SMTP パスワード
    - **[認証情報のダウンロード]** ボタンを押してCSV保存
    - ※このパスワードはこの画面でしか表示されません

---

# アプリ側での設定値まとめ

- **SMTPサーバー (Host):** `email-smtp.ap-northeast-1.amazonaws.com` （東京リージョンの場合）
- **ポート番号 (Port):** 587 (STARTTLS推奨) または 465 (SSL/TLS)
- **ユーザー名 (User):** 手順5で取得したSMTPユーザー名
- **パスワード (Pass):** 手順5で取得したSMTPパスワード
- **送信元アドレス:** 手順2で検証済みのメールアドレス
