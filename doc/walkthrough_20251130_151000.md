# Stripe決済機能の実装 (Ticket 8)

## 概要
店舗がサブスクリプション契約を行うためのStripe決済機能と、決済完了・変更・キャンセルを検知して店舗ステータスを更新するWebhookハンドラーを実装しました。

## 実装内容

### 1. 依存関係のインストール
- `stripe` パッケージをバックエンドにインストールしました。

### 2. 環境変数の設定
`.env` ファイルに以下の変数を追加する必要があります（`.env.example` は更新していませんが、実際の運用環境で必要です）。
- `STRIPE_SECRET_KEY`: Stripeのシークレットキー
- `STRIPE_WEBHOOK_SECRET`: Webhook署名検証用のシークレット
- `NEXT_PUBLIC_APP_URL`: フロントエンドのURL（成功/キャンセル時のリダイレクト先）
- `STRIPE_PRICE_BASIC`: BasicプランのPrice ID
- `STRIPE_PRICE_MICRO_HP`: Micro-HPプランのPrice ID

### 3. APIエンドポイント

#### A. チェックアウトセッション作成
- **URL**: `POST /api/payment/create-checkout-session`
- **Body**: `{ "storeId": "...", "planType": "basic" }`
- **機能**: Stripeのチェックアウトセッションを作成し、決済ページのURLを返します。`client_reference_id` に `storeId` を埋め込みます。

#### B. Webhookハンドラー
- **URL**: `POST /api/payment/webhook`
- **機能**: Stripeからのイベントを受け取り、店舗情報を更新します。
  - `checkout.session.completed`: 店舗ステータスを `ACTIVE` にし、サブスクリプションIDを保存します。
  - `customer.subscription.updated`: サブスクリプションの状態変更を反映します。
  - `customer.subscription.deleted`: サブスクリプションがキャンセルされた場合、ステータスを更新します。

### 4. ミドルウェア設定 (`config/middlewares.ts`)
- `strapi::body` ミドルウェアの設定を変更し、`includeUnparsed: true` を有効にしました。これにより、Webhookの署名検証に必要な生のボディ（Raw Body）にアクセスできるようになりました。

### 5. フロントエンド実装
- **APIクライアント (`frontend/lib/api.ts`)**: `createCheckoutSession` 関数を追加しました。
- **店主管理画面 (`frontend/app/admin/dashboard/page.tsx`)**: サーバーコンポーネントで店舗情報を取得し、クライアントコンポーネントに渡すように修正しました。
- **UIコンポーネント (`frontend/components/admin/DashboardClient.tsx`)**: 店舗ステータスが `ACTIVE` でない場合、ヘッダーに「本契約する」ボタンを表示するようにしました。クリックするとStripe決済画面へ遷移します。

## テスト方法（ローカル開発）
1. Stripe CLIを使用してWebhookを転送します: `stripe listen --forward-to localhost:1337/api/payment/webhook`
2. 出力されたWebhook Secretを `.env` に設定します。
3. 店主管理画面 (`/admin/dashboard`) にアクセスし、「本契約する」ボタンが表示されていることを確認します（店舗ステータスがACTIVE以外の場合）。
4. ボタンをクリックして決済を行い、完了後に管理画面に戻り、ボタンが消えている（またはステータスが更新されている）ことを確認します。
