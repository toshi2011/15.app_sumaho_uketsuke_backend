### ⚠️ iOS（iPhone / iPad）

| 機能 | 状態 | 備考 |
|------|------|------|
| 通知音 | ⚠️ 部分的 | 「業務開始」直後は安定、長時間放置で不安定 |
| バッジ更新 | ✅ 動作確認済み | 正確な件数が表示される |
| ポーリング | ✅ フォアグラウンドのみ | バックグラウンドでは停止 |
| Web Push通知 | ❌ 未実装 | HTTPS本番環境が必要 |

#### iOS制限事項

1. **AudioContextのsuspend問題**
   - iOSは省電力のため、ユーザー操作がないとAudioContextを停止
   - 解決策: 定期的に画面をタップ / Web Push通知に移行

2. **バックグラウンド制限**
   - PWAがバックグラウンドになるとJavaScript実行が停止
   - ポーリングも停止するため、画面ロック中は通知不可

3. **Badge API未サポート**
   - iOSではPWAアイコンにバッジ（数字）を表示できない

## 今後の実装予定

### Web Push通知（本番環境デプロイ後）

#### 必要な要件

| 要件 | 説明 |
|------|------|
| HTTPS | SSL証明書が必要 |
| ドメイン | 正式なURL（例: `https://yoyaku.example.com`） |
| VAPID鍵 | サーバー側で生成する公開鍵/秘密鍵ペア |
| バックエンド実装 | 新規予約作成時にプッシュ通知を送信 |

#### 実装手順（予定）

1. **VAPID鍵の生成**
   ```bash
   npx web-push generate-vapid-keys
   ```

2. **環境変数に設定**
   ```
   VAPID_PUBLIC_KEY=BPxxxxxxx...
   VAPID_PRIVATE_KEY=xxxxxxx...
   VAPID_SUBJECT=mailto:admin@example.com
   ```

3. **フロントエンド**
   - プッシュ通知の購読（Subscribe）処理
   - Service Workerでプッシュ受信処理（既に実装済み）

4. **バックエンド**
   - 予約作成時にプッシュ通知を送信
   - `web-push` ライブラリを使用

#### 期待される効果

- **バックグラウンドでも通知受信**
- **画面ロック中でも通知受信**（iOS 16.4+）
- **安定した音声通知**
