# Ticket: Implement Store Preview Modal

## 概要
店舗設定画面において、保存前に変更内容を確認できるプレビュー機能を実装しました。

## 実装機能
1. **プレビューボタン**: 設定画面ヘッダーに「プレビュー」ボタン（目のアイコン）を追加。
2. **全画面モーダル**: ボタン押下時に全画面モーダルで店舗ページを表示。
3. **リアルタイム反映**: フォームに入力中の未保存データをプレビューに即座に反映。
4. **コンポーネント共有**: 店舗詳細ページ（`/store/[storeId]`）のUIロジックを `StorePortalUI` コンポーネントとして分離し、プレビューと本番ページで共有。

## 技術的詳細
- **StorePortalUI**: `Store` オブジェクトをPropsとして受け取る再利用可能なコンポーネント。
- **データマッピング**: フォームの状態（State）を `Store` 型のオブジェクトに変換してコンポーネントに渡すことでプレビューを実現。
