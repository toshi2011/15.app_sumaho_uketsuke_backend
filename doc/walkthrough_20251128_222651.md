# ウォークスルー: 店舗プレビューモーダルの実装

店舗設定画面（`/admin/settings`）に、現在の設定内容を即座に確認できる「プレビュー機能」を追加しました。

## 実装内容

### 1. `StorePortalUI` コンポーネントの作成
既存の店舗詳細ページ（`/store/[storeId]`）の表示ロジックを `StorePortalUI` コンポーネントとして切り出しました。これにより、実際の店舗ページとプレビュー画面で全く同じUIコンポーネントを共有し、保守性を高めました。

### 2. 店舗詳細ページのリファクタリング
`frontend/app/store/[storeId]/page.tsx` を更新し、新しく作成した `StorePortalUI` を使用するように変更しました。

### 3. 設定画面へのプレビュー機能追加
`frontend/app/admin/settings/page.tsx` に以下の機能を追加しました：
- **プレビューボタン**: ヘッダーの「保存」ボタンの横に目のアイコンを追加。
- **全画面モーダル**: ボタン押下時に全画面でプレビューを表示。
- **リアルタイム反映**: フォームに入力中の値（未保存の変更を含む）をプレビューに即座に反映。

## 検証結果

自動テストにより、以下の動作を確認しました：

## 技術的なポイント
- **コンポーネント再利用**: `StorePortalUI` を作成することで、コードの重複を排除しました。
- **リアルタイムプレビュー**: DB保存前の状態をプレビューに渡すことで、ユーザーは保存前に見た目を確認できます。
- **画像プレビュー**: アップロード前の画像ファイルも `URL.createObjectURL` を使用してプレビュー可能です（※今回の検証ではテキスト反映を中心に確認）。
