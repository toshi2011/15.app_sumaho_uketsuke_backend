# 店舗一括インポートスクリプト改修 (Ticket 6)

## 概要
`backend/scripts/import-stores.js` を改修し、店舗データのインポート時に画像（ロゴ、カバー）の自動設定とサンプルメニューの作成を行うようにしました。また、データベーストランザクションを導入し、データの整合性を確保しました。

## 変更内容

### 1. ディレクトリ構成の整備
以下のディレクトリを作成し、シード画像を配置できるようにしました。
- `backend/seed-images/logos/`
- `backend/seed-images/covers/cafe/`
- `backend/seed-images/covers/restaurant/`
- `backend/seed-images/covers/salon/`
- `backend/seed-images/covers/other/`

### 2. CSV構造の更新
`stores.txt` に `category` カラムを追加しました。
- 値: `cafe`, `restaurant`, `salon`, `other`
- これにより、店舗のカテゴリに応じたカバー画像が選択されます。

### 3. スクリプトのロジック変更 (`import-stores.js`)

#### A. カバー画像のランダム選択
- CSVの `category` に基づき、対応するフォルダ (`backend/seed-images/covers/{category}/`) からランダムに画像を選択してアップロードします。
- 画像が見つからない場合は `other` フォルダから選択し、それでもない場合は `null` となります。

#### B. ロゴ画像の特定マッチング
- CSVの `logo_path` に指定されたファイル名が `backend/seed-images/logos/` に存在する場合、それをアップロードして設定します。
- ファイルが存在しない場合はスキップします（エラーにはなりません）。

#### C. サンプルメニューの自動作成
- 店舗作成後、以下の内容でメニュー項目（`MenuItem`）を1つ自動作成します。
  - **商品名**: 【サンプル】メニューを登録してみましょう
  - **価格**: 0
  - **説明**: 管理画面の「メニュー設定」から、あなたのお店の自慢のメニューに入れ替えてください。

#### D. トランザクション処理
- 1行（1店舗）の処理（重複チェック、ユーザー作成、画像アップロード、店舗作成、メニュー作成）を1つのデータベーストランザクション内で実行します。
- 途中でエラーが発生した場合、その店舗に関する変更はすべてロールバックされます。

## 実行方法
```bash
cd backend
node scripts/import-stores.js
```

## 注意事項
- 画像ファイルは事前に `backend/seed-images/` 配下の適切なフォルダに配置してください。
- `stores.txt` のヘッダーには必ず `category` を含めてください。
