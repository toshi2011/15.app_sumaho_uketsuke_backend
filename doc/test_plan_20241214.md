## ⚠️ サーバー再起動時のループ問題

### 症状
- `npm run develop` または `.\start-backend.ps1` 実行後、ターミナルが無限ループ状態になる
- 強制終了（Ctrl+C）が必要

### 考えられる原因

1. **distディレクトリの破損**
   - Windows/exFAT環境での既知の問題
   - **対処**: `backend/dist` フォルダを手動削除してから再起動

2. **スキーマ変更後のキャッシュ不整合**
   - 新しいフィールド（lunchDuration等）追加後
   - **対処**: `npm run build` を実行してから `npm run develop`

3. **TypeScriptコンパイルエラー**
   - store.tsの変更でエラーがある可能性
   - **対処**: ターミナルのエラーメッセージを確認

4. **ホットリロードの競合**
   - 複数のファイル変更が同時に検出されている
   - **対処**: サーバーを完全に停止してから再起動

5. **ポーリングによる無限ループ（フロントエンド起因）**
   - 下記ログのように、同じAPIが高速で連続して呼ばれている
   - onPollコールバックがrefreshReservationsを呼び、それがrefreshNotificationsを呼び、再度onPollが発火する循環参照の可能性

### 問題発生時のサーバーログ
```
[2025-12-14 02:17:34.234] http: GET /api/reservations?populate=*&... (35 ms) 200
[2025-12-14 02:17:34.291] http: GET /api/reservations?filters... (34 ms) 200
[2025-12-14 02:17:34.326] http: GET /api/reservations?filters...pending... (34 ms) 200
[2025-12-14 02:17:34.390] http: GET /api/reservations?populate=*&... (37 ms) 200
[2025-12-14 02:17:34.445] http: GET /api/reservations?filters... (34 ms) 200
[2025-12-14 02:17:34.482] http: GET /api/reservations?filters...pending... (36 ms) 200
[2025-12-14 02:17:34.531] http: GET /api/reservations?populate=*&... (34 ms) 200
[2025-12-14 02:17:34.589] http: GET /api/reservations?filters... (38 ms) 200
[2025-12-14 02:17:34.622] http: GET /api/reservations?filters...pending... (33 ms) 200
[2025-12-14 02:17:34.686] http: GET /api/reservations?populate=*&... (45 ms) 200
バッチ ジョブを終了しますか (Y/N)? Y
```

### 分析
- **50-60msごとに同じ3つのAPIが繰り返し呼ばれている**
- 通常のポーリング間隔は60秒のはずだが、これは明らかに異常
- 可能性：`onPoll` → `refreshReservations` → `refreshNotifications` → `pollNotifications` → `onPoll` の循環

### 要調査ファイル
- `frontend/hooks/usePWANotifications.ts` - onPollの呼び出しタイミング
- `frontend/components/admin/DashboardClient.tsx` - refreshReservationsの実装
